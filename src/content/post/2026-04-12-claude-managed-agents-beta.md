Anthropic 刚放了个大招。

今天他们正式推出了 **Claude 托管代理（Managed Agents）Beta 版**。说白了，就是 Claude 现在能在云端自己“住”在一个小容器里，然后**自动帮你读文件、跑命令、上网查东西、写代码**——而且全程不用你盯着。

你可能会说：“啊？那跟自己调 API 写 Agent 有啥区别？”

区别可太大了。

以前你想让 Claude 跑一个长任务，比如“去这个网站扒数据，存下来，再分析一下，画个图”，你得自己写循环、处理中断、管上下文、担心超时……折腾下来代码量不小，还容易崩。

现在呢？Anthropic 把这些都帮你包好了。你直接调 API，Claude 就在托管环境里**自己干活**，干完了叫你。而且它**专为长任务和异步设计**，可以跑几分钟甚至更久。

更贴心的是：你不是撒手不管。中间你可以随时**实时引导它或者打断它**——“停停停，不是这个意思，换一个方式。”

另外有个小细节挺良心：内置了**提示词缓存和性能优化**。这意味着你不用自己去搞那些加速技巧，它默认就挺快。

当然，刚出来肯定有限制。目前 API 有频率限制：**每分钟最多创建 60 次代理请求，读取最多 600 次**。对于个人开发者或者小团队来说完全够用了，大厂的话可能要等后续放开。

还有他们画了个饼：**多代理协作**和**长期记忆**现在只是“研究预览”阶段。也就是说，以后可能好几个 Claude 分工合作，一个查资料一个写代码一个测试，而且还能记住你昨天交代的事——想想还挺带感的。

一句话总结：如果你现在还在手撸 Agent 框架，可以歇歇了。Anthropic 这个托管代理，让你**直接喊 Claude 去干活**，不用再写那些又臭又长的循环代码了。

想玩的直接去他们官网看 API 文档。我已经准备试试了，看看它到底能帮我省多少时间。
