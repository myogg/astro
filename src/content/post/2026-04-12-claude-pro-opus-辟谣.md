**前两天网上突然炸了锅：有人说 Claude Pro 想在 Claude Code 里用 Opus 模型，得额外交钱。**

结果 Claude 的工程师 Thariq 直接现身说法：**假的，那是篇过期的旧文章。**

## 怎么回事？

事情是这样的：有人翻出了一篇 Anthropic 官方的支持文章，里面写着 Pro 用户要用 Opus 得单独买额度。

大家一看就慌了——我都订阅 Pro 了，凭啥还要再掏钱？

但 Thariq 很快发帖澄清：

> “这是一篇过时的支持文章，写于我们尚未在专业计划中提供 Opus 之时。我们已在一月为 Opus 4.5 推出该功能，但未更新这篇文章。对于造成的困扰，我们深表歉意。”

简单翻译一下就是：**那篇文章是旧的了，1 月份 Opus 4.5 发布的时候，Pro 就已经能用了，只是忘了把文章更新。**

## 那现在到底啥情况？

- **Claude Pro 订阅用户**：在 Claude Code 里直接用 Opus 4.5，**不需要额外花钱**
- 那篇引起恐慌的文章：已经被官方打上了“过期内容”的标记
- 工程师态度：道歉很诚恳，“给大家添麻烦了，不好意思”

## 一个小教训

以后看到“官方文档”说要加钱，可以先留意一下**发布日期**。

这次就是个典型：文章还在，功能早改了，搞得大家虚惊一场。

## 总结一句话

**Pro 用户放心用 Opus，不额外收费。工程师亲自盖章，不用慌。**

*如果你还想确认，可以直接去 Claude Code 里试一下模型选项——Opus 4.5 就在那里躺着等你用。*
